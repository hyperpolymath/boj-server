== Overview

**Bundle of Joy Server** is a Rhodium Standard Repository (RSR) compliant project providing a unified, formally verified catalogue of developer server capabilities. It was bootstrapped from `rsr-template-repo` and maintains all 17 mandatory workflows.

This document describes the project's RSR compliance posture and structure.

== Quick Start

[source,bash]
----
# Clone
git clone https://github.com/hyperpolymath/boj-server
cd boj-server

# Enter development environment (Guix primary, Nix fallback)
guix shell -D -f guix.scm
# or: nix develop

# Build all FFI layers
just build

# Type-check all ABI definitions
just typecheck

# Run tests
just test

# Full verification (zero believe_me + type-check + tests)
just verify
----

== Three-Layer Stack

Every cartridge follows the ABI/FFI universal standard:

[cols="1,1,3"]
|===
|Layer |Language |Purpose

|**ABI**
|Idris2
|Formal proofs, `%default total`, zero `believe_me`

|**FFI**
|Zig
|C-compatible native execution, zero runtime deps

|**Adapter**
|V-lang
|Triple API (REST + gRPC + GraphQL) on single port
|===

== RSR Compliance

=== Required Files (all present)

* `.editorconfig`
* `.gitignore`
* `Justfile`
* `README.adoc`
* `LICENSE` (PMPL-1.0-or-later)
* `.machine_readable/STATE.a2ml`
* `.well-known/security.txt`
* `.well-known/ai.txt`
* `.well-known/humans.txt`
* `guix.scm` (primary) + `flake.nix` (fallback)

=== Language Stack

* **ABI**: Idris2 (dependent types, formal verification)
* **FFI**: Zig (zero-cost C ABI, cross-compilation)
* **Adapter**: V-lang (triple API, Phase 3)
* **Infrastructure**: Just, Guile Scheme (`.a2ml` files), Nickel (k9), Shell

=== Prohibited (enforced by CI)

* TypeScript/JavaScript (use ReScript)
* npm, Bun, pnpm, yarn (use Deno)
* Python (use Julia/Rust)
* Go (use Rust/Zig)
* `Dockerfile` (use `Containerfile`)

=== 17 Mandatory Workflows

All present in `.github/workflows/`:

* `hypatia-scan.yml` — Neurosymbolic security scanning
* `codeql.yml` — Code analysis
* `scorecard.yml` — OpenSSF Scorecard
* `quality.yml` — Quality checks (TruffleHog, EditorConfig)
* `mirror.yml` — GitLab/Bitbucket mirroring
* `instant-sync.yml` — Forge propagation
* `guix-nix-policy.yml` — Guix/Nix enforcement
* `rsr-antipattern.yml` — RSR validation
* `security-policy.yml` — Security policy checks
* `wellknown-enforcement.yml` — .well-known/ validation
* `workflow-linter.yml` — Workflow validation
* `npm-bun-blocker.yml` — Package manager policy
* `ts-blocker.yml` — TypeScript blocker
* `jekyll-gh-pages.yml` — GitHub Pages
* `jekyll.yml` — Jekyll
* `scorecard-enforcer.yml` — Scorecard enforcement
